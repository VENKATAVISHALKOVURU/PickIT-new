# Contributing to PickIT 🚀

First off, thank you for considering contributing to PickIT! It's people like you that make PickIT such a great tool for students worldwide.

## 🌈 How Can I Contribute?

### Reporting Bugs
- Use the [GitHub Issue Tracker](https://github.com/VENKATAVISHALKOVURU/PickIT-new/issues).
- Describe the bug and provide steps to reproduce it.
- Include information about your environment (Browser, OS).

### Suggesting Enhancements
- Open an issue with the tag `enhancement`.
- Describe the feature you'd like to see and why it would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.

## 🎨 Style Guide
- We use **Prettier** for formatting.
- Follow the **Standard JS** style guide for TypeScript.
- Use descriptive commit messages (e.g., `feat: add student history`, `fix: header overlap`).

## 🛡️ Security
Please report security vulnerabilities to **security@pickit.app**. Do not open a public issue for security bugs.
