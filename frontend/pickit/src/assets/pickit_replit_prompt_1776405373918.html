Build a full-stack SaaS web app called PickIT — a print request platform for students and shop owners. === TECH STACK === Frontend: React + Tailwind CSS + Framer Motion Backend: Node.js + Express Database: PostgreSQL (use Prisma ORM) File storage: Firebase Storage or local multer for MVP Auth: JWT-based (bcrypt passwords) QR: Use qrcode.react (frontend) + uuid (backend) === PREMIUM UI REQUIREMENTS === - Use Framer Motion for ALL page transitions (fade+slide in) - Animate every card mount with staggered entrance (0.08s delay each) - Use spring physics on hover (scale 1.02, shadow lift) - Loading states: animated skeleton shimmer (CSS keyframes) - Status badges animate in with a pop (scale 0 → 1.05 → 1) - Dashboard stats count up from 0 on load (number counter animation) - Use a smooth morphing gradient background ONLY on landing/auth pages - All buttons have micro-press animation (scale 0.97 on click) - Toast notifications slide in from top-right with progress bar - File upload: drag-drop zone with ripple on drop - Use Inter font, clean white cards, subtle 1px borders === USER ROLES === 1. Shop Owner 2. Student === AUTH FLOW === - Landing page with animated hero - Register as Student or Shop Owner - After shop owner registers → auto-generate a unique UUID shop code - Generate a QR code from that UUID and display it on their dashboard - Students scan QR → redirected to /join/{shopCode} → pre-fills shop selection - Student links their account to that shop via QR scan === SHOP OWNER DASHBOARD === Pages: Overview, Orders, QR Code, Pricing, Analytics, Settings - Overview: animated stat cards (total orders, pending, revenue today) - Orders: real-time list with status chips (Pending / Printing / Ready / Done) - QR Code page: show large QR code, shop name, download as PNG button, copy link button - Pricing: set per-page prices (B&W, Color), minimum order, paper sizes - Analytics: recharts bar chart (orders/day last 7 days), line chart (revenue trend) - Settings: shop name, address, open/close toggle === STUDENT DASHBOARD === Pages: Upload, My Orders, History - Upload: drag-drop file (PDF/image), choose pages, color mode, copies, add note - Show estimated price from shop pricing config - My Orders: card list with status timeline animation - History: past orders === QR CONNECT FLOW (CRITICAL) === - Shop owner creates account → backend generates shopCode (UUID) - Backend generates and stores shopCode in DB under shop profile - On owner dashboard QR page: render - Student scans QR → opens /join/:shopCode in browser - If student not logged in → redirect to login/signup then back - After auth → link student to that shop → store shopId on student profile - Student's upload page now defaults to that shop === DATABASE SCHEMA === Tables: users, shops, orders, files, pricing_config - users: id, name, email, password, role (student|owner), shopId (nullable) - shops: id, ownerId, name, shopCode (UUID), address, isOpen - orders: id, studentId, shopId, fileUrl, pages, colorMode, copies, status, price, createdAt - pricing_config: id, shopId, bwPerPage, colorPerPage, updatedAt === API ROUTES === POST /auth/register POST /auth/login GET /shop/qr-info (owner gets their QR + shopCode) POST /shop/join/:shopCode (student links to shop) POST /orders/create GET /orders/student (student's orders) GET /orders/shop (owner's incoming orders) PATCH /orders/:id/status GET /analytics/orders (owner) === LANDING PAGE === -
